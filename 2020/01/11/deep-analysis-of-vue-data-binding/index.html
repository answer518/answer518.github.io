1、在做大型项目时，用哪个框架？

文无第一，武无第二， 技术选型没有对错！

vue本质是mvvm框架，由mvc发展而来。
react本质是组件化框架，由后端组件化发展而来。

两者都能实现大部分功能。

vue使用模板 - angular提出，指令等
react使用jsx

组件化

react天生就是为组件化支持的
vue是从mvvm发展而来，也支持组件化

相对来说，react组件化更清晰和彻底。

2、vue在大型项目中，容易出现内存泄漏的原因是什么？

滥用data

所有的响应式属性都会挂在统一的vue实例上面，如果组件卸载了，但是组件挂载在vue实例上的属性并不会销毁。这就是导致内存泄漏的第一个原因。

vue dep 维护依赖 Watcher(v-html, v-if) 用于连接视图层和响应式数据层的

MVVM view-model => Watcher

keep-alive

3、说一下vue的整体实现流程？

1、解析模板成为render函数

with, 变量绑定，v-click v-model 指令解析
render函数返回vnode

2、响应式开始监听

核心api: Object.defineProperty
将data属性代理到vm上

3、首先渲染页面，且绑定依赖

首次渲染，执行updateComponent, 中间会vm.render()
vm.render函数会访问到data属性，触发响应式get流程
updateComponent, 会走到vdom的patch方法
patch将vdom渲染成dom, 完成页面渲染

4、data属性变化，重新渲染页面

data属性发生变化，被响应式set监听
执行updateComponent方法
updateComponent会重新执行vm.render()
传入生成的vnode和prevVnode并执行patch方法
完成页面渲染

4、Object.defineProperty() 能不能监听到数组的变化？

看一个例子：

1、
Object.defineProperty 只能监测到对象一开始就存在的key. 对于数组只能监测数据下标的时候才会触发。

2、针对数组
push 一定不会触发，
unshift 可能会触发，
pop shift 可能会触发

vue2源码为什么会重写数组的监听和操作方法？

首先不是Object.defineProperty监听不到数组的变更，因为对于数组的每一次操作，都可能影响数组其它元素索引的变更。
